Terragrunt is a thin wrapper for Terraform that provides extra tools for keeping your Terraform
configurations DRY, working with multiple Terraform modules, and managing remote state.

Each terragrunt module you execute should have a hierarchy that follows this specification:

[0;m- ./[0m[0;36m$account_number[0m[0;m/[0m[0;36m$region_name[0m[0;m/[0m[0;36m$environment_name[0m[0;m/[0m[0;36m$module_name[0m

Parameters:

[0;32m- [0m[0;36m$account_number   [0m[0;32m: AWS account[0m
[0;32m                      Set with $AWS_ACCOUNT or --account "$account_number"[0m
[0;m[0m
[0;32m- [0m[0;36m$region_name      [0m[0;32m: AWS region name or "_global" for region-less services (such as IAM or Route53)[0m
[0;32m                      Set with $AWS_REGION or --region "$region_name"[0m
[0;32m                      Override with $HAL_METADATA_REGION ("region" build parameter)[0m

[0;32m- [0m[0;36m$environment_name [0m[0;32m: dev, test, beta, prod, etc (Do not use "-aws" suffix)[0m
[0;32m                      Automatically set with $HAL_ENV[0m
[0;32m                      Override with --environment "$environment_name"[0m

[0;32m- [0m[0;36m$module_name      [0m[0;32m: The name of the infrastructure module to run[0m
[0;32m                      Set with $HAL_METADATA_MODULE ("module" build parameter) or --module "$module_name"[0m

Other options:
- Prefix a value to "$account_number" with $AWS_ACCOUNT_PREFIX
- Prefix a value to "$environment_name" with $ENVIRONMENT_PREFIX

Destroying infrastructure:
- Set $HAL_METADATA_DESTROY to any non-empty value such as "1" ("destroy" build parameter)
  This will trigger a terragrunt destroy.

Running commands other than plan|apply|destroy:
- Run a custom terragrunt command by setting $HAL_METADATA_CUSTOM_COMMAND ("custom_command" build parameter)
- To run a custom command during deploy, set $HAL_METADATA_DEPLOY_COMMAND ("deploy_command" push parameter)

Adding targets to run commands against a subset of resources:
- Set $HAL_METADATA_PLAN_TARGET ("plan_target" build parameter) and Hal will automatically set -target="your_value".
- Use $HAL_METADATA_PLAN_TARGET_1 ("plan_target_1 build parameter") to set many targets (1 through 10).

------------------------------------------------------------------------------------------------------------------------

For multi-module projects $HAL_METADATA_MODULE is always required and should not be hardcoded.
- Set "module" as a required build parameter under Hal > Select Your Application > Manage Application > Edit Application
